google_results

When we get a result from the Google Custom Search Engine API we want to hold onto all the results until we use them up. For the time been we're going to store them into this variable in memory since our usage is low. Need to move this to a database in the future for scalablity reasons. Reason for this is due to a 100 query limit when for searching. Hopefully querying for 10 results should mean we have a limit of 1,000 images per API key
